# 枚举总表

## 物理字段约束

节点固定 6 列：

- `id`
- `label`
- `type`
- `description`
- `evidence_source`
- `en_identifier`

边固定 3 列：

- `source`
- `target`
- `interaction`

---

## 节点类型枚举

| type | 含义 | id 前缀 |
|---|---|---|
| Service | 业务 / 开放服务 | `svc_` |
| NetworkFunction | 网元 / 核心功能实体 | `nf_` |
| Interface | 参考点 / 接口面 | `iface_` |
| Protocol | 协议 / 鉴权协议 / 发现协议 | `proto_` |
| FQDNPattern | 域名命名模式 | `fqdn_` |
| Platform | 平台 / 框架 / 能力开放体系 | `plat_` |
| Capability | 具体开放能力 | `cap_` |
| APIArtifact | 开放服务产物 / API 侧对象 | `api_` |
| StandardDoc | 官方标准 / 官方规范文档 | `doc_` |
| RiskHypothesis | 风险假设 | `risk_` |
| WorkProduct | 系统产物 / 分析模块 | `wp_` |
| Metric | 度量指标 | `met_` |
| Vulnerability | CVE / 错误配置 / 标准实现缺陷（有证据） | `vuln_` |
| ThreatVector | 威胁载荷或攻击手法（有证据概括） | `tv_` |

补充规则：

- 只用 `FQDNPattern`，不用 `NamingRule`
- 只用 `StandardDoc`，不用 `Doc` / `Spec` / `Paper`
- 只用 `RiskHypothesis`，不用 `Risk`
- 只用 `NetworkFunction`，不用 `NF`
- 只用 `APIArtifact`，不再拆出新 type

## 边类型枚举

| interaction | 含义 |
|---|---|
| uses_network_function | 服务使用某网元 |
| exposes_interface | 服务或网元暴露某接口 |
| uses_protocol | 实体使用某协议 |
| uses_naming_pattern | 实体使用某域名模式 |
| depends_on | 依赖关系 |
| resolved_via | 通过某协议发现 / 解析 |
| implemented_via | 通过某平台实现 |
| uses_capability | 服务 / 平台使用某能力 |
| exposes_artifact | 暴露某 API 侧产物 |
| documented_in | 在某官方文档中被定义 / 描述 |
| governed_by | 受某规范 / 框架治理 |
| targets | 指向某风险假设 |
| has_component | 平台包含组件 |
| mitigated_by | 被某工作产物缓解 |
| produces | 产出某指标 |
| vulnerable_to | 某实体在证据支持下易受此弱点/手法影响 |
| enables_vector | 弱点或配置缺陷使某 ThreatVector 成为可能 |

---

## 边连接约束

| interaction | 允许的连接 |
|---|---|
| uses_network_function | `Service -> NetworkFunction` |
| exposes_interface | `Service -> Interface`；`NetworkFunction -> Interface` |
| uses_protocol | `Service -> Protocol`；`NetworkFunction -> Protocol`；`Interface -> Protocol`；`APIArtifact -> Protocol` |
| uses_naming_pattern | `Service -> FQDNPattern`；`NetworkFunction -> FQDNPattern`；`APIArtifact -> FQDNPattern` |
| depends_on | `Service -> Service`；`Service -> NetworkFunction`；`NetworkFunction -> NetworkFunction`；`Service -> Capability`；`APIArtifact -> Capability`；`APIArtifact -> APIArtifact` |
| resolved_via | `NetworkFunction -> Protocol`；`Service -> Protocol`；`APIArtifact -> Protocol` |
| implemented_via | `Service -> Platform`；`Interface -> Platform`；`Capability -> Platform` |
| uses_capability | `Service -> Capability`；`Platform -> Capability` |
| exposes_artifact | `Service -> APIArtifact`；`Platform -> APIArtifact`；`NetworkFunction -> APIArtifact` |
| documented_in | source 可为业务实体，target 必须是 `StandardDoc` |
| governed_by | `Service -> StandardDoc`；`Platform -> StandardDoc`；`Capability -> StandardDoc` |
| targets | `Service / NetworkFunction / Interface / Protocol / FQDNPattern / Capability / APIArtifact -> RiskHypothesis` |
| has_component | `Platform -> Platform`；`Platform -> Capability`；`Platform -> APIArtifact` |
| mitigated_by | `WorkProduct -> RiskHypothesis` |
| produces | `WorkProduct -> Metric` |
| vulnerable_to | `Protocol -> Vulnerability`；`Interface -> Vulnerability`；`NetworkFunction -> Vulnerability`；`APIArtifact -> Vulnerability` |
| enables_vector | `Vulnerability -> ThreatVector`（需证据链） |
