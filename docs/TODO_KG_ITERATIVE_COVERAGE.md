# 知识图谱迭代覆盖抽取方案（待办与实施计划）

> 状态：设计文档 / 实施待办清单（**尚未改业务代码**）。  
> 目标：在「朴素分块抽取」之上，用 **可度量的原文覆盖** 驱动 **多轮补抽**，并在 **预算与收敛判据** 下停机。

---

## 1. 目标与非目标

### 1.1 目标（必须达成）

- G1：在单文档（及可扩展到语料集）上，对 **已抽取三元组（节点+边）** 保留 **可追溯原文证据**（现有 evidence 机制上增强一致性校验即可）。
- G2：定义 **形式化的「原文覆盖」度量**，使「哪些正文尚未被当前图谱解释」可计算、可复现。
- G3：基于未覆盖区域触发 **补抽（仍可为朴素分块 LLM 抽取）**，并与 **检索（RAG）** 结合，形成 **有限轮迭代**，直至满足 **严格停机条件**。
- G4：全流程有 **审计日志**：每轮输入范围、检索 query、命中片段、模型版本、新增/拒绝的三元组、停机原因。

### 1.2 非目标（本阶段明确不做或降级）

- NG1：不承诺「完备抽尽」——在预算内 **最大化覆盖** 即可。
- NG2：不替换现有 Neo4j 合并语义为先验真理；**冲突**走单独「仲裁/降级」策略，不在首版实现复杂本体推理。
- NG3：首版 **不要求** 字符级 OCR 版面引擎；PDF 仍以现有文本提取为准（后续可插拔增强解析）。

---

## 2. 术语与形式化定义

### 2.1 文档与分块

- **原文 `D`**：单篇文档的规范化全文字符串（与当前 `chunk_text` 使用的文本一致）。
- **分块序列**：`C = {c_0, …, c_{n-1}}`，每块含 `(chunk_index, text, char_span)`。  
  - **首版强制**：`char_span = [start, end)` 在 `D` 上可验证（见 4.2）；若当前管线无 span，则 **P0 待办** 为「切块时记录 span」。

### 2.2 图谱增量状态

- **图谱快照 `G_k`**：第 `k` 轮结束后的节点/边集合（与 Neo4j 中 `Entity` 子图同构视图即可）。
- **新状态（用于停机）**：  
  - **强定义（推荐）**：`Δ_k = (G_k \ G_{k-1})` 在规范化后的 `(subject_id, predicate, object_id)` 集合上 **基数 > 0**（去重后）。  
  - **弱定义（可选）**：仅统计「新 evidence 指向此前未覆盖 span 的比例」> 阈值（更易抖，需配合强定义）。

### 2.3 证据与覆盖（核心）

- **证据引用 `e`**：来自抽取结果中的 `evidence`（`source_file`, `chunk_index`, `quote`）。  
- **quote 锚定（首版）**：在对应块文本 `c_i.text` 内对 `quote` 做 **子串定位**（若失败则记为 **无效证据**，不计入覆盖，并进入质检待办）。
- **块覆盖（P0 度量）**：块 `c_i` 被覆盖，当且仅当存在至少一条 **有效证据** 其 `chunk_index == i` 且 `quote` 在 `c_i.text` 中非空匹配。  
  - **覆盖率（块级）**：`cover_block = |{ i : c_i 被覆盖 }| / n`。
- **字符覆盖（P1 度量，更严）**：将每条有效证据的 `quote` 在 `D` 上映射到 span（通过 `char_span` 平移），得集合 `S ⊆ [0, |D|)`；  
  - `cover_char = |⋃ intervals(S) ∩ [0,|D|)| / |D|`（合并重叠区间后长度 / 全文长度）。  
  - **注意**：同一句多证据只计一次长度（区间合并）。

### 2.4 检索在方案中的角色（修正「对关系做 RAG」）

- **不**将「裸边字符串」作为唯一检索 query。  
- **Query 构造（必须文档化）**，首版采用 **Query Pack**：

  | Query 类型 | 内容模板 | 用途 |
  |-----------|----------|------|
  | Q1 | `实体标签 + 类型 + description 摘要` | 召回可能提及该实体的未覆盖段 |
  | Q2 | `source_id + predicate + target_id + 两端标签` | 召回可能陈述该关系的段落 |
  | Q3 | `从 G_k 抽样的「子图摘要」`（限长） | 主题级补漏 |

- **索引对象**：默认仍以 **chunk 向量** 为主（与现有 GraphRAG chunk 一致）；**必须**带 `metadata.source_file`、`chunk_index`（及未来 `char_span`）。

## 3. 输入 / 输出 / 前置条件

### 3.1 输入

- 已解析的 `D`、分块 `C`、文件标识 `source_file`。
- 现有规则文本 `R`（`data/rule` + `data/rules`）。
- 配置：模型名、每轮最大 LLM 调用数、最大轮数、覆盖阈值、最小新增边数等（见第 7 节）。

### 3.2 输出

- 最终 `G_final` 已合并入 Neo4j（或 staging 再人工合并，二选一需在实现前定案）。
- **覆盖报告**（每文档 JSON）：各轮 `cover_block`、`cover_char`、未覆盖块列表、`Δ_k` 大小、停机代码。
- **运行日志**（结构化）：检索 query 列表、top-k 文档 id、补抽块 id 列表。

### 3.3 前置条件（不满足则降级或中止）

- P0-A：`llm_enabled` 与 embedding 可用（若某轮需向量检索）。  
- P0-B：Milvus（或等价向量库）中 **已存在该 `source_file` 的 chunk 向量**；否则迭代前 **必须先跑 embed-only 入库**（与现有脚本对齐）。  
- P0-C：Neo4j 可写；`graph_backend=neo4j` 时全功能；为 file 后端时 **仅运行覆盖模拟 + 导出**，不声称生产就绪。

## 4. 算法流程（可实施步骤）

### 4.0 总览（轮次 `k = 0 … K_max-1`）

1. **Round-0（朴素）**：对 `C` 全量块跑现有抽取 → 合并得 `G_0`；写证据；算 `U_0`（未覆盖度量）。  
2. **若停机条件满足 → 结束**（见第 6 节）。  
3. **构造 Query Pack** 自 `G_k`（抽样 + 限流，避免 query 爆炸）。  
4. **向量检索**：对每个 query 取 top-`m` chunk，**过滤**：仅保留「当前未覆盖」块（块级或字符级二选一）。  
5. **合并候选块**：去重、限最大块数 `B_max`，必要时合并相邻未覆盖块（P1）以减少调用次数。  
6. **补抽**：仅对候选块跑朴素抽取 → 规范化合并 → `G_{k+1}`。  
7. **k ← k+1**，回到步骤 2。

### 4.1 Query 抽样（防止组合爆炸）

- 对节点：按 **度 / 出现频次 / 轮次新颖度** 排序，取前 `N_node` 个生成 Q1。  
- 对边：取前 `N_edge` 条生成 Q2。  
- `N_node`、`N_edge`、`m`、`B_max` **全部来自配置** 并记录日志。

### 4.2 char_span 改造要求（P0 阻塞项）

- `chunk_text`（或 PDF 后处理）在生成每块时 **同时输出** `[start,end)`，且满足 `D[start:end] == chunk_text`（规范化后逐字相等或在约定 normalize 后相等）。  
- 若无法实现（例如 PDF 提取与原文 offset 不一致），则 **首版仅允许块级覆盖**，并在文档封面标明 **不可启用字符级停机条件**。

### 4.3 无效证据处理

- `quote` 无法在块内定位 → 标记 `evidence_invalid`，该条不参与覆盖，**可选**触发该块进入补抽候选（配置开关，避免与正常「空块」冲突）。

---

## 5. 冲突、去重与合并（每轮必须）

- **节点 id**：沿用现有 `normalize_node_id`。  
- **边**：`(source, interaction, target)` 去重；若属性冲突 → **保留证据更强**（quote 更长 / 多源一致）或 **降低置信标记**（若 schema 扩展）。首版可 **后者不进库仅记日志**。  
- **与 Neo4j MERGE**：保持现有 `merge_nodes_edges` 语义；迭代轮次仅追加增量。

---

## 6. 停机条件（须同时配置默认值）

满足 **任一** 即停（可配置为 AND 组合，但需写明）：

- **S1（强）**：连续 `T_stall` 轮 `|Δ_k| == 0`（无新规范化三元组）。推荐默认 `T_stall = 2`。  
- **S2（覆盖）**：`cover_char ≥ θ_char`（若启用 char span）；否则 `cover_block ≥ θ_block`。推荐首版仅 S2 块级：`θ_block = 0.92`（可调）。  
- **S3（预算）**：累计 LLM chat 调用 ≥ `L_max` 或 美元等价预算（按项目统计）。  
- **S4（轮数）**：`k ≥ K_max`。推荐默认 `K_max = 5`。  
- **S5（退化）**：连续两轮检索候选块集合为空。

**停机记录**：必须输出 `stop_reason ∈ {STALL, COVERAGE, BUDGET, MAX_ROUNDS, NO_CANDIDATES, ERROR}`。

## 8. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 检索飘、引入无关块 | 强制过滤未覆盖 + 限制 top-m + query 限长 |
| 费用指数涨 | `B_max`、`L_max`、`K_max` 三重帽 |
| 假收敛（模型随机吐新边） | 强停机 S1 + 规范化三元组去重 + 可选「人工白名单类型才计数」 |
| PDF 无稳定 offset | 首版仅用块级覆盖；版面解析作为 P2 |
| 与现有全量入库并行 | **互斥锁或文档级队列**（实施待办） |

## 9. 验证与验收（测试计划摘要）

- **单元级**：span 一致性、`quote` 定位、区间合并、停机状态机。  
- **集成级**：单篇小 YAML（已有验证脚本同构）跑 2 轮，要求 `stop_reason` 合法、日志齐全。  
- **回归级**：迭代关闭时行为与当前「单遍朴素」逐位等价（配置开关默认 off）。
